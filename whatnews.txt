Ultimate Basic v1.4.4 – Release Notes
======================================

2026-06-14 – Mouse support + fixes
--------------------------------------------------
  - 1351 proportional mouse support via mouse_x(), mouse_y(), mouse_btn()
    Built-in mouse_read_helper assembly subroutine handles:
      CIA1 $DC00 capacitor charging, LDX#$67 delay, SID POT X/Y read,
      signed 7-bit delta computation, Y-axis EOR#$FF inversion,
      accumulated position tracking, and dual-sample per frame.
  - mouse_btn() reads $DC01 (control port 1) — bit0=left, bit1=right.
  - mouse_x() returns accum_x via JSR to helper; mouse_y() reads accum_y
    directly (avoids double-update per frame).
  - Word var assignment (sx: word = mouse_x()) sets both lo/hi bytes
    for proper $D010 MSB handling by the sprite command.
  - 9-bit X carry/borrow via accum_x_hi (INC/DEC on overflow).
  - Demo: examples/mouse_demo.ub — 1351 mouse pointer sprite with
    button feedback (left=red border, right=blue background).
  - Fixed: KERNAL IRQ touches $DC00 during keyboard scan → corrupts POT;
    demo disables CIA1 interrupts (poke $DC0D, $7F + asm { SEI }).
  - Fixed: helper RTS must return LDA accum_x (not leftover raw_y).
  - Fixed: mouse helper ZP zeroed before game loop (fill $02, 7, 0).
  - Fixed: helper init uses same charge_delay as do_delta for
    consistent POT readings (no initial delta jump).
  - Fixed: prev_raw only updated when delta != 0 (matches original).

Ultimate Basic v1.4.3 – Release Notes
======================================

Recent development notes (2026-06-12)
--------------------------------------------------

2026-06-12
  - Added `fn` keyword — functions with return values.
      fn square(x)       → 8-bit return (result in A register)
        return x * x
      end
      fn get_ptr(): word → 16-bit return (result in fn_ret_zp pair)
        return $C000
      end
      var s = square(9)          # fn call as expression
      var ptr: word = get_ptr()  # word return
  - Typed subroutine parameters are now preserved end-to-end:
      sub init_message(src:string)
        var c = src[i]     # string indexing → LDA (ptr),Y
      end
    The parser, AST, and codegen all propagate string, int, word,
    float annotations on sub/fn parameters.
  - Unused variable reporting: the build output now lists variables that
    are declared but never read (tracked via mark_used across all variable
    read paths — eval_expr, print, inc/dec, array get/set, string params).
  - New example: examples/function_demo.ub — demonstrates fn with multiple
    return paths, word params, and expression-context calls.

Ultimate Basic v1.4.2 – Release Notes
======================================

Recent development notes (2026-05-30 / 2026-05-31)
--------------------------------------------------

2026-05-31
  - Added parallax starfield example programs:
      examples/parallax.ub
      examples/parallax_demo.ub
      examples/text_scroll_demo.ub
  - Added smooth text-scroller demo using hardware horizontal fine scroll.
  - Added new scroll helpers for pixel-scrolling examples:
      scroll x n narrow   (set x fine-scroll and force 38-column mode)
      scroll x n wide     (set x fine-scroll and restore 40-column mode)
      scroll row R left   (shift one screen RAM row left by one character)
  - Reworked text-scroll example to use UltimateBasic commands instead of inline asm for
    38/40-column mode and row shifting.
  - Clarified documentation: semicolon is not a BASIC comment marker; it is an asm-source
    comment marker inside asm { } blocks. BASIC comments are # and rem.
  - Added focused tests for scroll x narrow/wide, scroll row left, and semicolon handling.

2026-05-30
  - Added graphics on block: 80x50 chunky block-pixel mode using a custom 4-pixel charset
    at $2800, with no bitmap RAM required.
  - Added plot4 x, y and plot4 erase x, y for setting and clearing block pixels.
  - Added circle4 x, y, r for midpoint circle outlines in 80x50 block-pixel mode.
  - Fixed gcls in block mode so it clears all screen/color RAM pages correctly; the old
    descending DEX/BPL loop failed for counts >= 128.
  - Added lowercase and uppercase commands for switching C64 charset mode via CHROUT.
  - Updated TENPRINT example to demonstrate lowercase charset mode.
  - Removed temporary test/demo files from tmp/.

New language features
---------------------

Variables & types
  - float type (Q8.8 fixed-point): var f: float = 3.5 / int(f) / print f ("N.DD")
  - array_word(N): 16-bit word arrays at $C000+
  - Compound assignments: and=  or=  xor=  shl=  shr=
  - Full 16-bit arithmetic on word vars: * / mod shl shr and or xor
  - inc/dec carry-aware for word variables

Control flow
  - for/next with optional step and continue
  - repeat/until (do-while)
  - select/case/else
  - gosub label / return
  - continue / break in all loop types

Print & formatting
  - spc(n), tab(n) in print
  - print at col, row, ...  (no trailing newline)
  - hex(n), bin(n), dec(n, width)
  - numstr var, addr  (8-bit int → 3-digit decimal string)

Math & string functions
  - sin(angle), cos(angle)  (0–255 lookup table)
  - rnd(n)  (rnd() mod n)
  - min, max, abs, sgn
  - val(s)  (decimal string → 8-bit int)
  - s[i]  (string character by index)

Screen & display
  - screen col, row, char [, color]  (direct screen/color RAM write)
  - display on / display off  (VIC DEN bit, independent of mode)
  - scroll x n / scroll y n  (hardware fine-scroll registers)
  - scroll x n narrow / scroll x n wide  (fine scroll + 38/40-column mode control)
  - scroll row R left  (single screen-row character shift for smooth scrollers)
  - lowercase / uppercase  (switch C64 charset mode)
  - cursor col, row

Bitmap graphics
  - graphics on multi  (multicolor bitmap 160×200, 4 colours/cell)
  - mplot x, y, color  (multicolor pixel; requires graphics on multi)
  - plot erase, plot xor, line, paint

Block graphics (80×50)
  - graphics on block  (chunky 80×50 mode; custom 4-pixel charset @ $2800; no bitmap RAM)
  - plot4 x, y / plot4 erase x, y  (set/clear a block pixel, x: 0-79, y: 0-49)
  - circle4 x, y, r  (draw clipped midpoint circle outlines in block-pixel coordinates)
  - gcls clears screen RAM $0400-$07FF + color RAM $D800-$DBFF in block mode
  - Fix: gcls block-mode no longer leaves the bottom ~6 rows uncleared
    (the old descending LDX #231/DEX/BPL loop ran only once for counts ≥128;
     replaced with forward INX/BNE page loops)

Sprites & charset
  - sprite_frame id, addr  (update data pointer at runtime)
  - sprdef id...end  (inline 63-byte shape + auto pointer; binary literals)
  - sprite multicolor / expand x/y / priority
  - charset base / chardef id...end  (custom character definitions)

SID & music
  - sid volume N / sid stop
  - load sid "file.sid" [, addr]  (compile-time embed; defines sid_init/sid_play)
  - music play [N]  (CIA1 timer A 50 Hz IRQ + wrapper; auto-installs handler)
  - music stop / pause / resume

Interrupts & timing
  - nmi handler / nmi_exit
  - cia_timer period, handler
  - irq_exit  (JMP $EA81)
  - delay N  (N PAL frames)
  - wait raster N

Error handling
  - onerr goto label  (KERNAL I/O error vector $0300/$0301; forward refs OK)

Memory & I/O
  - peek16 / poke16  (16-bit little-endian)
  - fill, memcopy, drawmem
  - reu stash / fetch / swap + reu_present()
  - load / save / open / close / print# / input
  - data / read

Ultimate 64
  - speed N / speed max / speed off
  - badlines on / off
  - turbo()

CLI
---
  - Binary renamed from ultimate-basic to ub
  - --d64 [file]  produce .d64 disk image
  - --add <file>  add extra files to .d64 (repeatable)
  - -v / --verbose  zero-page layout + hex dump
